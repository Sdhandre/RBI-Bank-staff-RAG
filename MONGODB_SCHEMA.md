# 🗄️ MongoDB Schema — `bank_staff_rag`

## Database: `bank_staff_rag`

---

## Collection 1: `sessions`

Stores chat session metadata.

| Field | Type | Description |
|---|---|---|
| `_id` | ObjectId | Auto-generated by MongoDB |
| `id` | string (UUID v4) | Public session identifier used in all API calls |
| `title` | string | Defaults to `"New Chat"`; auto-set to first 60 chars of first user message |
| `created_at` | string (ISO 8601) | Timestamp when session was created |
| `updated_at` | string (ISO 8601) | Timestamp updated on every new message |

### Example Document

```json
{
  "_id": "ObjectId(...)",
  "id": "b3a1f2c4-8e9d-4f7a-a123-000000000001",
  "title": "What is the leave policy for Grade B…",
  "created_at": "2026-04-26T23:10:00.000000",
  "updated_at": "2026-04-26T23:15:42.123456"
}
```

---

## Collection 2: `messages`

Stores individual chat messages linked to a session.

| Field | Type | Description |
|---|---|---|
| `_id` | ObjectId | Auto-generated by MongoDB |
| `id` | string (UUID v4) | Unique message identifier |
| `session_id` | string | Foreign key → `sessions.id` |
| `role` | string | Either `"user"` or `"bot"` |
| `content` | string | The message text |
| `sources` | array | List of source citations (`[]` for user messages) |
| `sources[].source` | string | PDF filename e.g. `"RBI_Leave_Policy.pdf"` |
| `sources[].page` | number or `"?"` | Page number within the PDF |
| `created_at` | string (ISO 8601) | Timestamp when message was saved |

### Example: User Message

```json
{
  "_id": "ObjectId(...)",
  "id": "a1b2c3d4-0000-0000-0000-000000000001",
  "session_id": "b3a1f2c4-8e9d-4f7a-a123-000000000001",
  "role": "user",
  "content": "What is the leave policy for Grade B officers?",
  "sources": [],
  "created_at": "2026-04-26T23:10:05.000000"
}
```

### Example: Bot Message

```json
{
  "_id": "ObjectId(...)",
  "id": "e5f6a7b8-0000-0000-0000-000000000002",
  "session_id": "b3a1f2c4-8e9d-4f7a-a123-000000000001",
  "role": "bot",
  "content": "Grade B officers are entitled to 30 days of earned leave...",
  "sources": [
    { "source": "RBI_Leave_Policy.pdf", "page": 14 },
    { "source": "Staff_Regulations.pdf", "page": 32 }
  ],
  "created_at": "2026-04-26T23:10:07.000000"
}
```

## Relationship

```
sessions  (1) ──────────── (many)  messages
    id                  ←→             session_id
```

---

## Recommended Indexes

```python
# Add these for production performance
sessions_col.create_index("id", unique=True)
sessions_col.create_index("updated_at")                              # sort by recency
sessions_col.create_index("title")                                   # for ?q= title search

messages_col.create_index("session_id")
messages_col.create_index([("session_id", 1), ("created_at", 1)])   # paginated history
```

## Query Patterns Used in the App

| Operation | Query |
|---|---|
| List all sessions (sorted by recent) | `sessions.find({}).sort("updated_at", -1)` |
| Search sessions by title | `sessions.find({"title": {"$regex": q, "$options": "i"}})` |
| Get messages for a session | `messages.find({"session_id": sid}).sort("created_at", 1)` |
| Get last 6 messages (for LLM history) | `list(cursor)[-6:]` |
| Auto-title on first message | `sessions.update_one({"id": sid}, {"$set": {"title": ..., "updated_at": ...}})` |
| Delete session + all its messages | `sessions.delete_one({"id": sid})` + `messages.delete_many({"session_id": sid})` |
